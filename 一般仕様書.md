# 契約書管理システム 一般仕様書

## 1. システム概要
本システムは、管理者と消費者の間で契約書の作成、紐づけ、同意、および異議申し立てなどの一連のプロセスをデジタル上でセキュアかつ効率的に管理するためのデスクトップアプリケーションです。

## 2. 目的
- 契約内容の透明性確保と消費者保護
- 契約締結プロセスの電子化による業務効率化
- セキュリティ技術（暗号化や録音）を用いた同意プロセスの厳格化と証拠保全

## 3. 主要機能

### 3.1 管理者機能
- 契約書の作成、編集、削除（重要契約オプションの指定）
- ユーザーアカウントの作成、編集、削除
- ユーザーに対する契約書の紐づけ、紐づけ解除
- ユーザーの対応状況（同意、拒否、異議申し立て）の確認
- 異議申し立ておよび契約打ち切りの対応（保留へのリセット、代理同意）
- 締結済み契約の署名データ、事前説明録音データ、本人確認書類の閲覧

### 3.2 ユーザー（消費者）機能
- 自身に紐づいた契約書の閲覧
- 契約に対する同意（自書署名を含む）
- 契約に対する拒否
- 契約内容に対する異議申し立て
- 同意済み契約の打ち切り（代理同意された場合のみ）
- 重要契約における事前説明の確認、録音、および本人確認書類の提出

## 4. 契約ステータス定義
- 未対応 (Pending): ユーザーによる対応待ち状態。
- 同意済み (Agreed): ユーザーが契約に同意し、署名を完了した状態。または管理者が代理同意した状態。
- 却下 (Rejected): ユーザーが契約を拒否した状態。
- 異議申し立て中 (Disputed): ユーザーが契約内容に異議を申し立てている状態。
- 打ち切り済み (Terminated): 契約が途中解約または打ち切られた状態。

## 5. 動作環境
- OS: Windows 10 / 11 (64-bit)
- ランタイム: .NET 8.0 Desktop Runtime
- アーキテクチャ: x64

## 6. セキュリティ要件
- ユーザー認証: PBKDF2による安全なパスワードハッシュ化
- 録音データ: SHA-256ハッシュの計算によるデータ改ざん検知
- 本人確認書類: 楕円曲線暗号 (ECC NIST P-256) と AES-256-GCM を用いたハイブリッド暗号化
- データ保存先: ローカルファイルシステム内の JSON ファイル群（DataStore）
